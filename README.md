# Ralph

Autonomous TDD development loop powered by Claude Code. Write a spec, Ralph builds it — test by test, story by story.

Works with **greenfield** projects (scaffolds from scratch) or **brownfield** (initialises in any existing codebase).

## Quick Start — Brownfield

For existing projects of any stack:

```bash
# 1. Initialise Ralph in your project
cd ~/projects/my-app
ralph init

# 2. Write a spec describing what you want built (plain markdown)
# 3. Generate the task list
ralph task-build my-spec.md my-plan

# 4. Review interactively in Claude Code
#    /task-review my-plan

# 5. Run the loop
ralph run 20 my-plan
```

`ralph init` auto-detects your stack (Bun, Node, Python, or generic). Override with `--stack <preset>`.

## Quick Start — Greenfield

Scaffolds a new Bun/TypeScript project:

```bash
./create-project.sh ~/projects/my-app my-tasks.json my-plan
cd ~/projects/my-app
# In Claude Code: /task-review my-plan
ralph run 20 my-plan
```

## The Task Pipeline

Ralph uses a two-stage task pipeline:

1. **`ralph task-build`** — Automated. Takes a markdown spec and generates a structured task list JSON (`tasks-<plan>.json`) through iterative refinement. Fixes mechanical issues: oversized stories (>5 AC), missing dependencies, ambiguous acceptance criteria, ID gaps. If it encounters decisions it can't make, it reports them with a course of action.

2. **`/task-review`** — Interactive. Run in Claude Code for a conversational review of the generated task list. Catches qualitative issues: story ordering, missing edge cases, scope creep, architectural concerns.

Think of `task-build` as the linter and `/task-review` as the code review.

### Writing a Spec

Your spec is plain markdown — describe what you want built. It can be a feature spec, a task list, a productionisation plan, or anything else. Be specific about end-states. `task-build` will translate it into testable user stories.

If the builder can't determine something from your spec, it flags it as a human-decision item in the output. Update your spec with the answers and re-run.

### Task List JSON Format

Generated by `task-build`. The top-level `storyPrefix` field is derived from the `project` name (first letter of each hyphen-separated word, uppercased — e.g. `deck-manipulation` → `DM`). Each story has: `id`, `title`, `description`, `acceptanceCriteria` (max 5), `priority`, `passes` (starts false), `dependsOn`, `notes`.

```json
{
  "id": "DM-001",
  "title": "Project scaffolding",
  "description": "Initialise project with framework, test runner, and type checking",
  "acceptanceCriteria": [
    "Running the dev server starts on a configurable port",
    "GET / returns HTTP 200",
    "Tests pass with at least one smoke test"
  ],
  "priority": "high",
  "passes": false,
  "dependsOn": [],
  "notes": ""
}
```

## CLI Reference

```
ralph init [--stack <preset>] [target-dir]    Initialise .ralph/ in an existing project
ralph task-build <spec.md> [plan] [max-iter]  Generate task list from a markdown spec
ralph run <iterations> [plan]                  Run the autonomous TDD loop
```

Available stack presets: `bun-typescript`, `node-typescript`, `python`, `generic`.

## How the Engine Works

Each iteration:

1. Generates a codebase snapshot (file tree, exports, test counts)
2. Spawns a fresh Claude `--print` session with the iteration prompt
3. Claude picks the next failing story and implements it via RED-GREEN-REFACTOR
4. Progress is logged; circuit breakers detect stalls or repeated failures
5. Loop continues until all stories pass or iterations are exhausted

The engine runs with fresh context per iteration — no conversation history accumulates.

## Project Layout

After `ralph init`, your project gets:

```
.ralph/
├── config.sh              # Stack config (test/lint/typecheck commands)
├── engine/                # ralph.sh, prompt.md, snapshot.sh
├── skills/tdd/            # TDD methodology files
├── specs/                 # Specs and generated task lists
├── hooks/                 # Git safety hooks
├── progress.txt           # Append-only progress log
├── CLAUDE-ralph.md        # Ralph instructions for Claude
└── logs/                  # Iteration logs
```

## Testing

TDD is mandatory. RED-GREEN-REFACTOR. Vertical slices only. See `skills/tdd/SKILL.md` for the full methodology.

Infrastructure tests (no API calls): `tests/run-tests.sh`
